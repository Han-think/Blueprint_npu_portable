Method

HOW THIS BLUEPRINT WAS BUILT · for handoff & future iteration

1 · What this package is

19개 HTML 문서 모음으로 구성된 항공우주 추진기관(NPU) 디자인 패키지입니다. 정적 spec sheet가 아니라 cross-link 된 살아있는 문서 시스템 — 각 페이지가 React + inline JSX로 렌더되고, 공통 TopNav · 공통 색/타입 토큰으로 시각적 통일성을 유지합니다.

한 문장 요약: "AS9100 흐름을 따르는 추진기관 설계·제조·운영의 전 라이프사이클 (PDR → CDR → FAI → hot-fire → refurb) 을 정적 HTML로 가시화."

2 · Architecture

File layout

colors_and_type.css        ← design tokens (palette · fonts · base body class)
TopNav.jsx                 ← shared top navigation (cluster grouping, active state)
<PageName>.html            ← 19 documents, each self-contained
README.html                ← index, reading paths, conventions
Method.html                ← this document

Per-page skeleton

모든 페이지가 같은 뼈대를 따릅니다 — 새 페이지 추가 시 그대로 복붙해 사용:

<!DOCTYPE html><html lang="en"><head>
  <meta charset="utf-8">
  <title><PageName> · Blueprint NPU</title>
  <link rel="stylesheet" href="colors_and_type.css">
  <style>/* page-local styles only */</style>
</head><body class="bp">
  <div id="topnav"></div>
  <div id="root"></div>
  <!-- React 18.3.1 + Babel 7.29.0, pinned with integrity hashes -->
  <script src="https://unpkg.com/react@18.3.1/..."></script>
  <script src="https://unpkg.com/react-dom@18.3.1/..."></script>
  <script src="https://unpkg.com/@babel/standalone@7.29.0/..."></script>
  <script type="text/babel" src="TopNav.jsx"></script>
  <script type="text/babel">
    setTimeout(()=>window.BPMountTopNav && window.BPMountTopNav('<PageName>'), 100);
  </script>
  <script type="text/babel">
    /* page-specific React component & data */
    ReactDOM.createRoot(document.getElementById('root')).render(<App/>);
  </script>
</body></html>
WHY INLINE JSX (NOT MODULES)

각 파일이 외부 도구 없이 브라우저에서 바로 열림. CDN 의존성만 있으면 됨. 파일을 통째로 카피해서 새 페이지를 만들기 쉬움. 단점: 컴포넌트 재사용은 window globals를 통해서만 가능 — TopNav만 공유되고 나머지는 페이지 내부에 인라인.

Shared TopNav contract

TopNav.jsx가 window.BPMountTopNav(activeKey) 글로벌 함수를 export합니다.

  • 네비게이션 항목은 파일 상단 배열에 선언 — 각 항목은 {k, href, label, group}
  • group 값이 같은 항목들끼리 시각적으로 묶이고, 그룹 사이에 세로 구분선이 자동 삽입
  • 현재 페이지는 activeKey로 식별 — 색·border로 강조
  • 새 페이지 추가 = TopNav.jsx 배열에 한 줄 추가 + 새 HTML 작성

이름은 LOX/LH2/intertank 같은 로켓 용어에서 따옴 — 색의 "역할"을 추론하기 쉽게. 새 페이지에서도 C 객체로 동일 토큰을 인라인 정의해 일관성 유지.

Typography

  • Inter: 본문 · 라벨
  • JetBrains Mono (.mono): 수치, ID, 좌표, 시간, 코드 — "측정된 것"을 표현
  • 크기: 9.5px (mono meta) · 11–13px (본문) · 16–22px (KPI 숫자) · 28px (페이지 타이틀)
  • letter-spacing: 라벨에 .08–.14em — 청사진/도면 느낌

Layout conventions

  • 1200px max-width 컨테이너, 24px gutter
  • 패널: 1px solid frame 보더 + c2 배경 + 14px 패딩
  • 주요 패널은 좌측에 3px 컬러 보더 (semantic accent)
  • 그리드 + flex로 모든 레이아웃 구성, 인라인 흐름 지양

4 · Process — how we got here

Phase 1 · Scope & research

"항공기/엔진 어셈블리"를 단순 부품 트리가 아닌 AS9100 흐름으로 해석. GD&T (ASME Y14.5-2018), 공차 stack-up (worst-case vs RSS), 항공우주 체결 카탈로그 (Hi-Lok · V-band · lockbolt), LPBF 결함 (witness line · 40° overhang · HIP), AS9100 traceability를 사전 조사.

Phase 2 · Skeleton

  1. colors_and_type.css + TopNav.jsx 먼저 만들고 1페이지로 검증
  2. Assembly/Parts/Materials 3종으로 시각 언어 확정 (블루프린트 dark)
  3. 이후 모든 페이지가 이 시각 언어를 상속

Phase 3 · Iterate by cluster

한 번에 한 클러스터씩 추가, 사용자 승인 후 다음 클러스터:

  1. STRUCTURE: Assembly · Joints · Mass — 물리적 형상
  2. MATERIAL: Materials · PrintQueue · Inspection · Supply — 공급망 + 제조
  3. ANALYSIS: Loads · Thermal · Test · FMEA — 검증
  4. OPS: Lineage · Ops · ECN · ICD — 변경 · 수명 · 인터페이스
  5. PROGRAM: Schedule · Risk · Dashboard — 한 단계 줌아웃

Phase 4 · Capstone

Dashboard가 모든 페이지를 한 화면에 요약하는 capstone. README가 reading paths (persona별 추천 동선)를 정리. 본 Method 문서가 빌드 매뉴얼.

모든 페이지가 같은 가상 SN-002 빌드 컨텍스트를 공유합니다 — 숫자가 페이지 간에 일치:

새 페이지 추가 시 이 값들과 정합성 유지 — Dashboard와 Schedule이 가장 먼저 틀어집니다.

6 · Adding a new page

  1. 위 per-page skeleton 그대로 복사해 NewPage.html 생성
  2. 페이지 내부에 인라인 C palette 객체 복사 (다른 페이지 참조)
  3. 데이터는 페이지 상단 const 배열로 — 직접 편집 가능하도록
  4. React 컴포넌트는 App 하나로, 분리 최소화
  5. TopNav.jsx NAV 배열에 항목 추가, 적절한 group 지정
  6. README의 reading paths에 자연스럽게 등장하면 cross-link 추가
  7. Dashboard에 KPI 1개 추가 고려 (있으면)

7 · Known limitations

  • 모든 데이터가 하드코드 — JSON 또는 CSV 분리하면 더 maintainable
  • 페이지 간 cross-link은 정적 — 한 페이지에서 데이터 변경 시 수동 sync 필요
  • 인쇄 스타일(@media print) 미적용 — PDF 익스포트 시 색 조정 필요
  • 국문/영문 혼용 — 라벨은 영어, 본문 일부는 한글. 단일 언어로 정리 권장
